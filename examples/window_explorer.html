Catching a change before it catches you

A pump's seals are slowly wearing. The level sensors read normal — until, one day, they don't. How early can an algorithm call it?

The idea

A detector watches its own surprise

Fit a model of the plant's normal behaviour, then let it predict the next measurement. The gap between prediction and reality is the residual — small noise while the plant behaves, a jump the moment it changes and the model is caught off guard.

To turn that into an alarm we compare a recent test window against an older reference window:

score = max( Dtest / Dref − 1 , 0 )

where D is the mean residual over a window. The score sits near zero when the windows agree and spikes when the test window slides into the change. That spike is the detection.

Try it

Slide the windows across the change

It is a rolling-window statistic: two windows slide over the residual, and the score is the ratio of a recent test window to a local reference window.

The fast, recent view. Short → the statistic peaks sooner but is noisier; long → it peaks later but steadier. The peak lands ~one test-window after the change, so the detection delay ≈ the test length.

The local baseline. A fast wiggle that isn't a real change sits in both windows and cancels in the ratio. Short → adapts fast, hugs the recent normal; long → a steadier anchor that calms a twitchy test window.

The gap between them — slides the reference earlier while the test window stays put. Short → reference right behind test; long → baseline taken from further back, ignoring more of the recent past.

The trade-off

Where do you set the alarm?

A threshold turns the score into a yes/no alarm. Set it too low and you cry wolf — a high FAR; too high and you miss the event — a high MAR. Drag the threshold; the curve sweeps every threshold at once.

The threshold-free summary is the AUC in the table: higher is better, and it doesn't depend on where you set the bar.

Make it harder

Real changes aren't clean steps

Slow drifts and noisy sensors are what actually break detectors. Make the transition gradual and turn the noise up — the delay grows and the AUC falls.
