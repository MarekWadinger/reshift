    cpdmd_compare ¶

    Adapter + shared evaluation utilities for the CPDMD baseline comparison.

    Compares ODMD-CPD (this repo) against the external CPDMD baseline of V. Khamesi, "Online Changepoint Detection in Multivariate Seasonal Time Series via Dynamic Mode Decomposition", https://github.com/vkhamesi/cpdmd-python

    The baseline's source is used unmodified. Notebooks clone it on demand into examples/.baselines/cpdmd-python (gitignored); :func:ensure_cpdmd performs the clone and puts it on sys.path. This module only wraps the baseline so it consumes the same data layout and is scored by the same metric as ODMD-CPD.

    Functions:

    • causality_test –

      Demonstrate the CPDMD baseline is causal (no look-ahead) for delay 0.

    • cpdmd_error_series –

      Per-step normalized reconstruction error of the baseline's windowed DMD.

    • ensure_cpdmd –

      Clone the CPDMD baseline on demand and add it to sys.path.

    • gen_seasonal –

      Bivariate seasonal signal with per-segment frequency and amplitude.

    • gen_steps –

      Bivariate piecewise-constant signal with evenly spaced level steps.

    • guideline_params –

      Derive ODMD-CPD parameters via the paper's systematic selection.

    • latency –

      Per-sample wall-clock cost of ODMD-CPD vs the CPDMD baseline.

    • match_changepoints –

      Greedily match predictions to ground truth in a causal forward window.

    • metrics_table –

      Tabulate match metrics for both methods on one benchmark.

    • peaks –

      Collapse a score curve into discrete detections.

    • plot_bess –

      Plot BESS temperatures and score with the fault region shaded.

    • plot_comparison –

      Plot the signal and ODMD-CPD score with both methods' detections.

    • robustness –

      Multi-seed mean +/- std for ODMD-CPD, CPDMD, and a random detector.

    • run_cpdmd –

      Run the CPDMD baseline and return detected change-point indices.

    • run_odmd_cpd –

      Stream X through the ODMD-CPD detector and return scores + detections.

    causality_test ¶

    causality_test(
    tau: int = 1000,
    n: int = 1400,
    window: int = 200,
    order: int = 10,
    rank: int = 2,
    burn_in: int = 700,
    sigma: float = 0.1,
    jumps: tuple[float, ...] = (3.0, 1.0, 0.5, 0.2),
    seed: int = 0,
) -> tuple[DataFrame, DataFrame]

    Demonstrate the CPDMD baseline is causal (no look-ahead) for delay 0.

    Runs the baseline's single_change on data progressively truncated around tau: if it detects the change only once the array extends past tau (and the firing window reads no sample beyond tau), the detection is causal, not written a posteriori. A second sweep over jump magnitudes shows the delay grows as the change becomes subtler -- i.e. delay reflects accumulated evidence, and delay 0 only occurs for abrupt, large jumps.

    Parameters:

    • tau ¶

      (int, default: 1000 ) –

      True change index.

    • n ¶

      (int, default: 1400 ) –

      Sequence length for the jump sweep.

    • window ¶

      (int, default: 200 ) –

      CPDMD window.

    • order ¶

      (int, default: 10 ) –

      CPDMD time-delay order.

    • rank ¶

      (int, default: 2 ) –

      CPDMD rank.

    • burn_in ¶

      (int, default: 700 ) –

      CPDMD burn-in (must be < tau to allow detection).

    • sigma ¶

      (float, default: 0.1 ) –

      Noise std for the truncation sweep.

    • jumps ¶

      (tuple[float, ...], default: (3.0, 1.0, 0.5, 0.2) ) –

      Jump magnitudes (as absolute level shifts) for the delay sweep.

    • seed ¶

      (int, default: 0 ) –

      RNG seed.

    Returns:

    • DataFrame –

      (truncation_df, jump_df). truncation_df maps the highest data

    • DataFrame –

      index available to the detected index; jump_df maps jump size (in

    • tuple[DataFrame, DataFrame] –

      sigmas, using the jump sweep's own noise) to detected index and delay.

    match_changepoints ¶

    match_changepoints(
    true_cps: list[int],
    pred_cps: list[int],
    tol: int,
    pre: int = 0,
) -> dict[str, float]

    Greedily match predictions to ground truth in a causal forward window.

    A prediction p is a true positive for change t only if it lands in [t - pre, t + tol] -- i.e. at the change or after it (a causal detector cannot fire before the change manifests). pre allows a few samples of grace for windowing/indexing jitter; with pre=0 any pre-change detection is a false positive, so reported delays are non-negative and measured on true positives only.

    Parameters:

    • true_cps ¶

      (list[int]) –

      Ground-truth change-point indices.

    • pred_cps ¶

      (list[int]) –

      Detected change-point indices.

    • tol ¶

      (int) –

      Forward detection window (max admissible delay, in samples).

    • pre ¶

      (int, default: 0 ) –

      Samples of pre-change grace for jitter (default 0).

    Returns:

    • dict[str, float] –

      Dict with tp, fp, fn, precision, recall, f1, mean_delay (mean over true

    • dict[str, float] –

      positives, >= -pre; NaN if none).
